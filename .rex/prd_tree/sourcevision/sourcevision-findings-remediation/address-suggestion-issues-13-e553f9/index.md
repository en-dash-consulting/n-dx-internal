## Summary

- Add a subsection to CLAUDE.md's Architecture section documenting the web package's internal zone layering: web-shared (foundation) → viewer-messaging-stack (messaging middleware) → web-viewer (consumer), with web-server as a parallel composition-root tier. The four-tier monorepo hierarchy is documented but the web package's own internal stack is not, leaving developers without a documented model for intended import direction within the package.
- Three web-package zones (web-server: 4 files, web-shared: 3 files, logo: 2 files) are below a meaningful minimum zone size for health metric reliability. Cohesion and coupling calculations on fewer than 5 files produce high-variance readings that change dramatically with single-file additions. Define a minimum zone size threshold of 5 files below which health metrics are suppressed or annotated as unreliable in zone health reports.
- 3 zones exceed architectural risk thresholds (cohesion < 0.4, coupling > 0.6): request-deduplication, viewer-message-pipeline, packages-rex:validate — mandatory refactoring recommended before further development
- hench-agent spans 31 directories with no sub-zones, while the web package applies sub-zone decomposition at roughly 60 files per sub-zone. At 155 files, hench-agent qualifies for at least 2-3 sub-zones (e.g. agent-core, store, prd-integration, guard-policy). Without sub-zones, any zone-level coupling finding has a 155-file blast radius and directory-level architectural drift is invisible to sourcevision health metrics.
- The deleted .sourcevision/zones/mcp-route-layer/ directory in the git status (shown as D in the diff) confirms that 'mcp-route-layer' was previously an explicit zone ID for domain-gateway.ts and routes-mcp.ts. The 4-file sub-zone should be re-pinned under 'mcp-route-layer' to restore the semantically correct zone name that was lost.
- All existing zone insights for request-deduplication that describe 'two implementations', 'split consumer bases', and a 'causally linked atomic change' are based on the false premise that shared/request-dedup.ts contains an independent implementation. Update these insights to reflect the actual state: one implementation (viewer/messaging/), one dead shim (shared/), one correct test. The remediation complexity has been significantly overstated.
- packages/web/src/shared/request-dedup.ts is dead code: it re-exports from viewer/messaging/request-dedup.ts, and shared/index.ts already re-exports the same symbols directly from the same canonical source. No source file imports from shared/request-dedup directly. Delete the shim file in a single commit — no consumer updates, no test changes, no import path redirects are needed.
- Zone "Request Deduplication" (request-deduplication) has critical risk (score: 0.67, cohesion: 0.33, coupling: 0.67) — requires refactoring before new feature development
- Add mcp-transport.test.js and integration-coverage-policy.test.js to CLAUDE.md's Key Files table alongside architecture-policy.test.js and domain-isolation.test.js. All four files form the project's architectural enforcement corpus — omitting two of them from the Key Files table creates an incomplete picture of enforcement coverage for developers consulting documentation.
- Zone 'viewer-message-pipeline' is the sole zone with both high coupling (0.65) AND low cohesion (0.35) simultaneously — a fragile combination where changes have wide blast radius and internal structure provides no stabilizing guidance. This risk profile warrants priority resolution before adding any new files that import from this zone, as the coupling score will increase further. Resolve by zone-pinning the misclassified build scripts to web-dashboard and viewer utilities to web-viewer, then verify coupling drops below 0.3.
- Zone "Viewer Message Pipeline" (viewer-message-pipeline) has critical risk (score: 0.65, cohesion: 0.35, coupling: 0.65) — requires refactoring before new feature development
- Zone "Analyzers 3" (packages-sourcevision:analyzers-3) has catastrophic risk (score: 0.73, cohesion: 0.27, coupling: 0.73) — requires immediate architectural intervention
- Zone "Validate" (packages-rex:validate) has critical risk (score: 0.64, cohesion: 0.36, coupling: 0.64) — requires refactoring before new feature development
